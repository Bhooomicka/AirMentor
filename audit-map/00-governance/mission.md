# Mission

Build and maintain a deterministic audit operating environment for a near-lossless deconstruction of AirMentor across code, runtime behavior, roles, data flows, state transitions, ML heuristics, testing gaps, UX friction, and live deployment drift.

## Non-Negotiables

- No shallow passes.
- No reliance on ephemeral chat memory.
- No assumption that local equals live.
- No silent model or provider switching.
- No unattended long-running job outside detached `tmux`.
- No correctness sacrifice for token savings on high-risk work.

## Required Outcomes

- Feature, sub-feature, and micro-interaction maps
- Role-surface maps
- Dependency, data-flow, and state-flow maps
- ML/inference component audit
- Test and verification gap audit
- UX and complexity audit
- Live-vs-local reconciliation with evidence
- Durable ledgers for coverage, contradictions, and assumptions
