# Caveman Safety Policy

- Caveman may be used for low-risk summaries, ledger maintenance, terse reviews, and optional memory compression.
- Caveman must not be forced on for high-stakes synthesis, security-sensitive reasoning, ML semantics, live-vs-local contradictions, or final reports.
- Every run that uses Caveman must record that fact in status or checkpoint output.
- The default state is off.
