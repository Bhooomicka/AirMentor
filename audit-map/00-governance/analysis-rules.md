# Analysis Rules

- Treat every surface as four separate questions: expected, implemented, tested, live.
- Record route-level, component-level, and action-level evidence separately.
- Link findings to files, commands, artifacts, or snapshots.
- Update ledgers before ending a pass.
- When uncertain, create a checkpoint and stop safely instead of bluffing.
- Prefer bounded repeatable passes over broad undocumented exploration.
- Prefer file-backed summaries over pasting giant raw outputs into future prompts.
