# UX Friction Pass Prompt v2.0

Objective: identify concrete complexity, friction, ambiguity, hidden workload, and cognitive load in the scoped UX.

Required outputs:

- UX friction entries using `templates/ux-friction-template.md`
- workflow-level and control-level friction notes
- role-specific asymmetry notes
- evidence-backed consequences, not vague taste judgments

Rules:

- tie every friction claim to a concrete surface, task, control, state, or branching decision
- assess label clarity, discoverability, density, restore-state confusion, hidden coupling burden, unclear proof or ML framing, edit/save/lock clarity, and unnecessary steps
- be explicit about what likely confuses a smart user and why

Completion gate:

- every important scoped workflow must have either a friction assessment or an explicit note that no meaningful friction was evidenced
